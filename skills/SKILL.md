# Skill 安全审查流程

## 何时触发

在安装任何第三方 skill 之前，**必须先执行本流程**。只有通过审查的 skill 才能安装。

## 审查步骤

### 1. 读取所有文件

读取 skill 目录下的**每一个文件**，包括隐藏文件。不要只看 SKILL.md。

```bash
find <skill-dir> -type f -name '.*' -o -type f | head -50
```

### 2. 检查红旗（任一命中即拒绝）

| 红旗 | 说明 |
|------|------|
| `curl`/`wget` 到未知 URL | 可能外传数据 |
| `base64` 编码/解码 | 常见混淆手段 |
| `eval`/`exec` 执行动态代码 | 注入风险 |
| 修改 `~/.ssh/`、`~/.gitconfig` | 篡改凭据 |
| 读取环境变量中的密钥 | `$API_KEY`、`$TOKEN` 等 |
| 访问 `openclaw.json` | 可能窃取配置 |
| 写入系统目录 | `/usr/`、`/etc/` 等 |
| 二进制文件 | 无法审查内容 |
| 过大的文件（>100KB 单文件） | 可能藏代码 |

### 3. 评估风险等级

| 等级 | 标准 | 行动 |
|------|------|------|
| 🟢 LOW | 纯 Markdown/文本，无脚本 | 可直接安装 |
| 🟡 MEDIUM | 有脚本但逻辑清晰，无网络访问 | 说明风险，请用户确认 |
| 🔴 HIGH | 命中任一红旗 | **拒绝安装**，报告具体问题 |

### 4. 输出报告

```
## Skill 审查报告：<skill-name>

- **来源**：<URL/仓库>
- **文件数**：N 个
- **风险等级**：🟢/🟡/🔴
- **发现问题**：（如有）
  - [ ] 问题描述
- **建议**：安装 / 需确认 / 拒绝
```

## 审查豁免

以下来源可跳过完整审查（仅做快速扫描）：
- OpenClaw 官方 skill（`openclaw/skills` 仓库）
- 用户自己编写的 skill

## 红线

- 审查期间**只读不执行**——不运行 skill 中的任何脚本
- 发现 HIGH 风险**立即停止**，不要继续分析其余文件
